The Technology

What is MCP?

Model Context Protocol (MCP) is an open standard that connects AI assistants to external tools and platforms. It allows Claude Code to read from Figma, write to Webflow, and keep everything in sync automatically.

F

Figma MCP

Connected

Reads design files, extracts components, pulls design tokens, and maps Figma components to production code. Authenticated via Figma's official OAuth with Enterprise Full seat access.

W

Webflow MCP

Connected

Manages the production website: pages, elements, styles, components, CMS collections, and scripts. Operates in read-only mode by default with explicit permission required for any writes.

Safety First

Governance & Controls

Every interaction is governed by strict access controls. The system operates in read-only mode by default and requires explicit permission for any modifications to the production site.

Access Controls

🔒
Read-Only Default

Webflow MCP operates in read-only mode. Write operations require explicit user approval for each action.

🛡
Permission Hooks

Pre-execution hooks validate every tool call before it runs. Credential leaks, SSR patterns, and production modifications are caught automatically.

👁
Full Audit Trail

Every MCP tool call is logged with timestamps, parameters, and results. Complete visibility into what was read, written, or modified.

Protection Layers

✅
Credential Detection

Automatically blocks commits containing API keys, tokens, or passwords. Prevents accidental exposure of sensitive data.

🔒
Production Safeguards

Production code and knowledge graph data cannot be modified without explicit approval. Changes are validated before execution.

🛠
Pattern Validation

SSR patterns, server-side code, and framework anti-patterns are detected and flagged before they reach production.

Getting Started

Setup Guide

Step 1: Connect Figma MCP ▼

Run this command to add the official Figma MCP server via Claude Code's native HTTP transport:

claude mcp add --transport http figma https://mcp.figma.com/mcp

A browser window will open for OAuth authentication. Approve access with your Figma Enterprise account. The connection persists across sessions.

Step 2: Connect Webflow MCP ▼

The Webflow MCP connects via a workspace API token configured in Claude Code's settings:

"webflow": { "command": "npx", "args": ["-y", "webflow-mcp-server"], "env": { "WEBFLOW_TOKEN": "your-webflow-api-token" } }

For Designer tools, open the Webflow Designer in your browser and ensure the MCP app is running.

Step 3: Create Code Connect Mappings ▼

Share a Figma file URL with Claude Code. It will analyze the components, scan the Webflow site, and suggest 1:1 mappings between design and production components.

// Example: Map Figma Button to Webflow Button Link { "nodeId": "1:243", "componentName": "Button/Primary", "source": "webflow://Button Link", "label": "Web Components" }
Step 4: Verify Connections ▼

Type /mcp in Claude Code to see all connected servers. Both Figma and Webflow should show as active (green). The Figma server authenticates via OAuth; the Webflow server connects via API token.
